# 技术文档写作规范

## 适用场景
当用户需要撰写 API 文档、用户手册、架构设计文档、README 等技术文档时应用此规范。

## 文档类型与结构

### API 文档
1. 概述：接口用途和适用场景
2. 请求：URL、方法、参数、请求体
3. 响应：状态码、返回格式、示例
4. 错误码：常见错误及处理建议

### 用户手册
1. 快速开始：5 分钟上手指南
2. 功能详解：按功能模块组织
3. 常见问题：FAQ
4. 更新日志：版本变更记录

### 架构设计文档
1. 背景与目标
2. 系统架构图
3. 核心模块设计
4. 数据模型
5. 接口设计
6. 部署方案

## 写作规范
- 使用简洁明了的语言，避免歧义
- 代码示例必须可运行、可复制
- 使用一致的术语表
- 图表辅助说明复杂概念
- 保持文档与代码同步更新

## 常见问题
- 文档过时：建立文档与代码的同步机制
- 示例不完整：确保代码示例包含必要的上下文
- 结构混乱：先写大纲再填充内容
